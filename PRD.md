# 荣耀冲突 (Honor Clash) - 产品需求文档 (PRD)

## 1. 产品概述

### 1.1 产品名称
荣耀冲突（Honor Clash）

### 1.2 产品定位
基于王者荣耀英雄IP的卡牌对战游戏，融合王者峡谷世界观卡牌玩法与炉石传说法力值系统，支持在线对战、卡组编辑、排行榜等功能。

### 1.3 目标用户
- 王者荣耀玩家
- 卡牌游戏爱好者
- 休闲游戏用户

### 1.4 核心价值
- 利用已有130个王者荣耀英雄数据，快速构建可玩的卡牌游戏
- 独特的英雄专属技能卡系统，增加策略深度
- 王者峡谷世界观沉浸式体验
- 部署在 Cloudflare 全球 CDN，访问速度快

### 2.4 对战系统

#### 2.4.1 对战模式
- **1v1 人机对战**（DEMO版本）
- 玩家选择英雄 vs AI随机选择英雄
- AI使用简单随机出牌策略

#### 2.4.2 游戏规则

**初始状态：**
- 双方英雄血量 = 英雄定义的HP值
- 初始手牌 = 5张
- 初始法力值 = 1

**回合流程：**
1. 回合开始
2. 法力值回满 + 上限+1（最大10）
3. 从牌库抽1张牌
4. 出牌阶段（可出任意张牌，只要法力值足够）
5. 结束回合
6. 切换到对方回合

**卡牌使用：**
- 点击手牌中的卡牌
- 如果是攻击牌，点击敌方英雄为目标
- 如果是防御/治疗/增益牌，自动对自身使用
- 法力值不足时卡牌灰显不可点击

**伤害计算：**
```
最终伤害 = 基础伤害 + 攻击加成 - 护甲值
最终伤害 = max(0, 最终伤害)
```

**胜负判定：**
- 一方英雄血量 ≤ 0 → 对方胜利
- 双方同时归零 → 平局

## 3. 非功能需求

### 3.1 性能
- 页面首屏加载 < 3秒
- API响应时间 < 500ms
- 游戏操作延迟 < 100ms

### 3.2 兼容性
- 现代浏览器（Chrome/Firefox/Safari/Edge最新2个版本）
- 响应式设计（桌面端优先，兼容平板）

### 3.3 安全
- 密码 bcrypt 加密存储
- JWT Token 认证
- API 输入校验
- SQL 注入防护（D1参数化查询）

### 3.4 可用性
- 界面清晰直观
- 操作反馈及时（按钮点击、伤害数字飘字）
- 战斗日志可追溯

### 3.5 动效系统
- **卡牌出牌动画**：卡牌飞出并显示技能名称
- **伤害飘字**：造成伤害时数字向上飘出
- **治疗飘字**：恢复血量时绿色数字飘出
- **英雄抖动**：受到伤害时英雄头像震动
- **血条变化**：平滑过渡动画+低血量闪烁警告
- **法力条变化**：平滑过渡动画
- **手牌进出**：卡牌加入/离开手牌时的缩放动画
- **游戏结束弹窗**：淡入淡出+弹跳效果

## 4. 数据需求

### 4.1 英雄数据来源
从 `data/heroes/info/*.json` 读取，提取：
- id, name, title, hero_type_name
- survive_value → 转换为游戏血量
- attack_value → 转换为游戏攻击力
- skills → 提取技能名称和描述

### 4.2 英雄头像来源
从 `data/heroes/images/*.jpg` 复制到前端静态资源

### 4.3 数据转换公式
```
游戏血量 = survive_value × 2 + 10
  (范围：16-30)

游戏攻击力 = attack_value
  (范围：1-10)
```
