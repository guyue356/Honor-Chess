## 数据库 Schema

### D1 建表 SQL

```sql
-- 用户表
CREATE TABLE IF NOT EXISTS users (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  username TEXT UNIQUE NOT NULL,
  password_hash TEXT NOT NULL,
  nickname TEXT NOT NULL,
  avatar_url TEXT DEFAULT '',
  created_at TEXT DEFAULT (datetime('now'))
);

-- 英雄表
CREATE TABLE IF NOT EXISTS heroes (
  id INTEGER PRIMARY KEY,
  name TEXT NOT NULL,
  title TEXT NOT NULL,
  class TEXT NOT NULL,
  hp INTEGER NOT NULL,
  attack INTEGER NOT NULL,
  skill_name TEXT NOT NULL,
  skill_desc TEXT NOT NULL,
  skill_effect TEXT NOT NULL,
  image_url TEXT NOT NULL
);

-- 卡牌表
CREATE TABLE IF NOT EXISTS cards (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  name TEXT NOT NULL,
  type TEXT NOT NULL,
  hero_id INTEGER,
  mana_cost INTEGER NOT NULL,
  effect TEXT NOT NULL,
  description TEXT NOT NULL,
  FOREIGN KEY (hero_id) REFERENCES heroes(id)
);

-- 用户卡组
CREATE TABLE IF NOT EXISTS user_decks (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  user_id INTEGER NOT NULL,
  hero_id INTEGER NOT NULL,
  name TEXT NOT NULL DEFAULT '默认卡组',
  card_ids TEXT NOT NULL,
  is_active INTEGER DEFAULT 0,
  created_at TEXT DEFAULT (datetime('now')),
  FOREIGN KEY (user_id) REFERENCES users(id),
  FOREIGN KEY (hero_id) REFERENCES heroes(id)
);

-- 战绩记录
CREATE TABLE IF NOT EXISTS battle_records (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  user_id INTEGER NOT NULL,
  hero_id INTEGER NOT NULL,
  opponent_hero_id INTEGER NOT NULL,
  result TEXT NOT NULL,
  rounds INTEGER NOT NULL,
  damage_dealt INTEGER DEFAULT 0,
  damage_taken INTEGER DEFAULT 0,
  played_at TEXT DEFAULT (datetime('now')),
  FOREIGN KEY (user_id) REFERENCES users(id)
);

-- 排行榜
CREATE TABLE IF NOT EXISTS leaderboard (
  user_id INTEGER PRIMARY KEY,
  wins INTEGER DEFAULT 0,
  losses INTEGER DEFAULT 0,
  win_rate REAL DEFAULT 0,
  total_damage INTEGER DEFAULT 0,
  updated_at TEXT DEFAULT (datetime('now')),
  FOREIGN KEY (user_id) REFERENCES users(id)
);
```

## 风险与应对

| 风险 | 影响 | 应对措施 |
|------|------|----------|
| 游戏平衡性差 | 影响体验 | 先上线再根据数据调整数值 |
| AI太弱/太强 | 影响体验 | 实现多档AI难度 |
| 卡牌效果有bug | 影响功能 | 详细的单元测试 |
| D1性能不足 | 影响响应 | 游戏状态存前端，D1只存持久化数据 |
| 部署配置复杂 | 延迟上线 | 提前准备部署脚本 |

### 快速启动命令

```bash
# 前端
cd frontend
npm install
npm run dev

# 后端
cd backend
npm install
npm run dev

# 部署
cd backend
wrangler d1 create honor-chess-db
wrangler d1 execute honor-chess-db --file=./src/db/schema.sql
wrangler d1 execute honor-chess-db --file=./src/db/seed.sql
wrangler deploy

cd ../frontend
npm run build
wrangler pages deploy dist --project-name=honor-chess
```
